BKT — 단원별 숙달도 추적

Bayesian Knowledge Tracing으로 "무엇을 알고 있는지" 확률적 추론

Probabilistic Model
Core Parameters
Concepts
P(L₀) = 0.1 Initial Learning
사전 학습 확률. "아직 배우지 않았으므로 10% 정도만 알고 있을 것이다."
P(T) = 0.1 Transition
학습 전이 확률. "한 문제를 풀어서 깨달음을 얻을 확률은 10%이다."
P(G) = 0.25 Guess
추측 확률. "모르지만 찍어서 맞힐 확률 (4지선다 기준)."
P(S) = 0.1 Slip
실수 확률. "알고 있지만 실수로 틀릴 확률."
types/bkt.ts
TS
interface KnowledgeState { conceptId: string; // 4가지 핵심 파라미터 pLearn: number; // P(L) - 현재 알고 있을 확률 pTransit: number; // P(T) - 학습 전이 pGuess: number; // P(G) - 추측 pSlip: number; // P(S) - 실수 // 숙달 단계 masteryLevel: | 'not_started' // < 0.1 | 'attempted' // < 0.5 | 'familiar' // < 0.8 | 'proficient' // < 0.95 | 'mastered'; // >= 0.95 practiceCount: number; }
function updateKnowledgeState( state: KnowledgeState, isCorrect: boolean ): KnowledgeState { const { pLearn, pTransit, pGuess, pSlip } = state; // Step 1: 관찰 결과(정답/오답)를 반영한 사후 확률 계산 let pLearnGivenEvidence: number; if (isCorrect) { // 맞힌 경우: (알고 맞힘) / (알고 맞힘 + 찍어서 맞힘) const pCorrect = pLearn * (1 - pSlip) + (1 - pLearn) * pGuess; pLearnGivenEvidence = (pLearn * (1 - pSlip)) / pCorrect; } else { // 틀린 경우: (알지만 실수) / (알지만 실수 + 몰라서 틀림) const pIncorrect = pLearn * pSlip + (1 - pLearn) * (1 - pGuess); pLearnGivenEvidence = (pLearn * pSlip) / pIncorrect; } // Step 2: 학습 전이 적용 (모르던 것도 이번에 배웠을 수 있음) const newPLearn = pLearnGivenEvidence + (1 - pLearnGivenEvidence) * pTransit; return { ...state, pLearn: newPLearn, ... }; }
