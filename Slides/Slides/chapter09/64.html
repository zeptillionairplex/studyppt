모든 조건을 조합한 최종 선택 로직 (안전장치 + ZPD + 약점 + 목표 정답률)

function selectAdaptiveQuestion( studentElo: number, studentKnowledgeMap: Map<string, KnowledgeState>, questionPool: Question[], config: SelectionConfig ): Question { // Step 1: 안전장치 필터 (교육과정 천장 + 블룸 상한) let candidates = questionPool.filter(q => q.eloRating <= config.maxDifficulty && q.bloomLevel <= config.bloomLevelMax && !q.isCompetitionLevel // 경시대회 문제 제외 ); // Step 2: ZPD 범위 — 맞힐 확률 55~85% 필터 candidates = candidates.filter(q => { const expected = expectedScore(studentElo, q.eloRating); return expected >= config.successRateRange[0] && expected <= config.successRateRange[1]; }); // Step 3: 약점 단원 우선 (BKT < 0.8) if (config.preferWeakTopics) { const weakTopics = [...studentKnowledgeMap.entries()] .filter(([_, state]) => state.pLearn < 0.80) .map(([topicId]) => topicId); const weakQ = candidates.filter(q => weakTopics.includes(q.topicId)); if (weakQ.length > 0) candidates = weakQ; } // Step 4: 목표 정답률 70%에 가장 가까운 문제 정렬 const targetDifficulty = studentElo - 400 * Math.log10(1 / config.targetSuccessRate - 1); candidates.sort((a, b) => Math.abs(a.eloRating - targetDifficulty) - Math.abs(b.eloRating - targetDifficulty) ); // Step 5: 상위 5개 중 랜덤 (다양성 확보) const topN = candidates.slice(0, Math.min(5, candidates.length)); return topN[Math.floor(Math.random() * topN.length)]; }

Config Interface
TS
interface SelectionConfig { targetSuccessRate: number; // 기본 0.70 successRateRange: [number, number]; // [0.55, 0.85] maxDifficulty: number; // 교육과정 천장 bloomLevelMax: number; // 블룸 상한 (4~6) preferWeakTopics: boolean; // 약점 우선 여부 }
Target Difficulty Calculation
MATH
TD = S - 400 × log₁₀(1/P - 1)

TD: Target Difficulty (목표 난이도)

S: Student Elo (학생 레이팅)

P: Target Probability (목표 정답률, 0.7)

Top-N Random Strategy
Logic
// 왜 상위 1개가 아니라 Top-5 랜덤인가? 1. 다양성: 같은 실력일 때 매번 같은 문제 방지 2. 오차 보정: Elo 추정치의 불확실성 고려 3. 콘텐츠 소모 방지: 특정 문제만 반복 노출 방지 const topN = candidates.slice(0, 5); return topN[Math.random() * 5];
