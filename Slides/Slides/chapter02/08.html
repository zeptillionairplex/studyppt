데이터 파이프라인 흐름도

원문이 문제가 되기까지의 6단계 여정 (Data Journey)

PostgreSQL 중심 데이터 아키텍처
Standard Pipeline
01

원문 저장 / 버전 생성

TipTap JSON을 원본으로 저장하고, 새로운 버전을 생성 (Rollback 가능)

table: document_versions
02

Annotation 추출

JSON을 파싱하여 모든 Mark(빈칸, 핵심개념)를 DB 인덱스로 추출

Parser Logic
03

파생 콘텐츠 생성

규칙(빈칸) + AI(MCQ, 서술형)를 결합하여 문제 초안 생성

AI Worker / BullMQ
04

선생님 검수 (HITL)

생성된 Draft 상태의 문제를 승인/반려/수정 (최종 품질 확정)

status: 'approved'
05

적응형 제공

학생 Elo 레이팅과 문제 난이도를 매칭하여 최적 문항 선별

Target Acc: 55~85%
06

피드백 루프

풀이 결과 수집 → 학생 레이팅 업데이트 + 문제 난이도 재조정

BKT Model Update
Exception Flow
Trigger: 원문 수정 발생

Diff 계산

이전 버전 vs 현재 버전 비교

영향범위 식별

변경된 Annotation 찾기

Stale 마킹

관련 문제셋 is_stale = true

알림 발송

관리자에게 "검토 필요" 알림
