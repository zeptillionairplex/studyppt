아키텍처 핵심 결정 9가지

의사결정 매트릭스 — 최종 선택과 핵심 근거
# 결정 사항 (Decision) 최종 선택 (Selected) 핵심 근거 (Rationale)
1 에디터 (Editor) TipTap (ProseMirror) 커스텀 Mark를 통한 의미론적 태깅(Semantic Tagging) 구현 용이
2 메인 DB PostgreSQL JSONB 네이티브 지원으로 문서 저장과 관계형 데이터 조인을 동시에 해결
3 원문 저장 형식 TipTap JSON → JSONB 구조화된 SQL 쿼리 가능 (단순 문자열 저장이 아님)
4 파생물 관리 패턴 핵심패턴 B (SSOT + 캐시) 일관성(SSOT)과 성능(캐싱)의 균형점. AI 재생성 비용 최소화.
5 캐스케이드 방지 Version 귀속 + 스냅샷 원문 변경/삭제 시에도 이미 생성된 문제 세트가 깨지지 않음
6 AI 모델 전략 LLM Router 단순 작업은 Gemini Flash(저가), 복합 추론은 Claude Sonnet(고가) 분기
7 AI 호출 방식 OpenAI 호환 REST SDK 종속성 제거, 환경변수 변경만으로 모델 교체 가능
8 버전 관리 (VC) 핵심PG Table + jsondiffpatch Git 라이브러리 대비 복합 SQL 쿼리 유연성 우월 (예: 특정 단원 내 Stale 문제 찾기)
9 학생 적응형 Elo + BKT + 천장제어 정확한 수준 측정과 교육과정 범위 이탈 방지(선행 학습 폭주 제어)
