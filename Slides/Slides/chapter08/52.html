버전 관리 DB 스키마 설계

브랜치, 자동 저장, Diff 캐시 및 저장 전략

PostgreSQL Schema
Table: document_branches
SQL
CREATE TABLE document_branches ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), document_id UUID REFERENCES documents(id) ON DELETE CASCADE, name VARCHAR(100) NOT NULL, -- 'main', 'exam-prep' description TEXT, is_default BOOLEAN DEFAULT FALSE, parent_id UUID REFERENCES document_branches(id), base_version_id UUID REFERENCES document_versions(id), UNIQUE(document_id, name) );
Table: document_autosaves
SQL
CREATE TABLE document_autosaves ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), document_id UUID REFERENCES documents(id) ON DELETE CASCADE, branch_id UUID REFERENCES document_branches(id), user_id UUID REFERENCES users(id), content_json JSONB NOT NULL, saved_at TIMESTAMPTZ DEFAULT NOW(), -- 사용자/문서/브랜치당 최신 1개만 유지 (UPSERT Key) UNIQUE(document_id, branch_id, user_id) );
Table: version_diffs
SQL
-- 버전 간 diff 결과 캐싱 (jsondiffpatch delta) CREATE TABLE version_diffs ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), from_version_id UUID REFERENCES document_versions(id), to_version_id UUID REFERENCES document_versions(id), diff_json JSONB NOT NULL, created_at TIMESTAMPTZ DEFAULT NOW(), UNIQUE(from_version_id, to_version_id) );
Logic: Auto vs Official Save
FLOW
자동 저장 (Auto Save)
Trigger: 5초 Debounce
Target: document_autosaves
Action: UPSERT (항상 1행 유지)
No Annotation Extract
No Stale Check
정식 저장 (Official Save)
Trigger: "저장" 버튼 클릭
Target: document_versions
Action: INSERT (버전 번호 증가)
Annotation 추출 & Diff 계산
영향받는 문제 Stale 표시
