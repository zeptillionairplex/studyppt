캐스케이드 방지 — 4가지 안전장치

원문 수정·삭제 시 파생물을 보호하는 전략

source_version_id 격리

모든 파생물은 특정 원문 버전에 귀속됩니다. 최신 원문이 업데이트되어도, 기존 문제는 생성 당시의 버전을 참조하므로 내용이 변하지 않습니다.

SQL Logic

SELECT * FROM quiz_sets
WHERE source_version_id = 'v1_uuid';
-- 원문이 v2가 되어도 이 문제는 v1을 참조

original_text_snapshot 보관

각 문제는 생성 시점의 원문 텍스트와 문맥을 자체적으로 저장합니다. 원문이 완전히 삭제되더라도 문제는 독립적인 개체로서 기능합니다.

Schema Design

quiz_questions.original_text_snapshot
= '광합성은 이산화탄소와 물을...'
-- 원문 의존성 제거 (Decoupling)

Soft Delete + 유예 기간

'삭제' 요청 시 즉시 데이터를 지우지 않고 플래그만 변경합니다. 30일간의 복원 유예 기간을 두어 실수로 인한 데이터 손실을 방지합니다.

Update Query

UPDATE documents
SET is_archived = TRUE, deleted_at = NOW()
WHERE id = $doc_id;

변경 영향 분석 (Diff)

원문 수정 시 이전 버전과의 차이(diff)를 계산하여, 실제 변경된 태그와 연관된 문제만 '재검토 필요(stale)' 상태로 표시합니다.

Backend Logic

const affectedTags = diff(v1, v2);
if (affectedTags.includes(quiz.tagId)) {
  quiz.is_stale = true;
}
