시스템 동작 흐름

입력(Input) 처리(Process) 출력(Output)
Step 01. 입력 선생님 / 입력자

TipTap 에디터

  • 교과서, 강의록 등 원문 작성
  • 의미론적 태그 부여

태깅 예시:

#빈칸_지정 #핵심_개념 #출제_영역
JSON 원본 저장 (Single Source)
Step 02. 처리 백엔드 + AI

AI 문제 생성 파이프라인

  • 태그 파싱 및 문맥 분석
  • LLM (Gemini/Claude) 활용
  • 다양한 유형 문제 초안 생성

선생님 검수 (HITL)

  • AI 생성물 승인 / 반려 / 수정
  • 최종 품질 보증 (QA)
Step 03. 출력 학생 / 교사

다양한 학습 뷰(View)

  • 빈칸 채우기 워크시트
  • 객관식/서술형 시험지
  • 개념 정리 카드 (Flashcards)

적응형 학습

  • Elo 레이팅 기반 난이도 조절
  • 단원별 숙달도(BKT) 추적
핵심 원칙: 원문이 수정되면, 관련된 모든 파생 콘텐츠에 '검토 필요(Stale)' 상태가 표시됩니다.
